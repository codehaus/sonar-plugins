Des méthodes protégées qui ne sont jamais utilisées par aucune classe du même projet correspondent à du code mort. Le code mort représente un code inutile et inopérant qui devrait être supprimé. Cela simplifie la maintenance en diminuant la taille du code maintenu, rendant plus facile la compréhension du programme et évitant l'introduction de nouveaux bugs.

Dans les cas suivants, Sonar ne considère pas les méthodes protégées comme du code mort :

  • Méthodes protégées qui redéfinissent une méthode d'une de leur classe mère.
  • Méthodes protégées d'une classe abstraite.
