Des méthodes privées qui ne sont jamais exécutées correspondent à du code mort. Le code mort représente un code inutile et inopérant qui devrait être supprimé. Cela simplifie la maintenance en diminuant la taille du code maintenu, rendant plus facile la compréhension du programme et évitant l'introduction de nouveaux bugs.

Dans les cas suivants, Sonar ne considère pas les méthodes privées comme du code mort :

  • Constructeur vide privé, intentionnellement créés pour interdire l'instanciation d'une classe.
  • Methods privées : readObject(...), writeObject(...), writeReplace(...), readResolve(...) qui peuvent être utilisées lorsque la classe implémente l'interface Serializable.
