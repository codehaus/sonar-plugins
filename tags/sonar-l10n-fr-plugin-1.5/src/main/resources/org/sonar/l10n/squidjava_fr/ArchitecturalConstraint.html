Un code source est conforme à un modèle architectural quand il adhère pleinement à un ensemble de contraintes architecturales. Une contrainte permet de refuser les références entre classes en utilisant des expressions.

    Vous pouvez par exemple utiliser cette règle pour :
  • interdire l'accès aux classes **web** depuis les classes **.dao.**
  • interdire l'accès à java.util.Vector, java.util.Hashtable et java.util.Enumeration pour toute classe
  • interdire l'accès à java.sql.** depuis les classes **.ui.** et **.web.**
